Deployment Infrastructure

AWS Cloud Guide

A detailed walkthrough for hosting your Credit Default Prediction API on industrial-grade cloud services.

01

Registry Initialization

We begin by setting up a Private Registry in Amazon ECR. This acts as the host for your application's deployment packages.

✓
Navigate to ECR Log in to your AWS Console and search for Elastic Container Registry.
✓
Create Repository Select "Create Repository", set Visibility to Private, and name it credit-default-prediction.
02

The "Push" Process (The Bridge)

This is how you move your local code to the AWS cloud. It involves 4 specific commands provided by Amazon.

✓
Find the Commands Inside your repository page, look at the top right for a button that says View push commands. Click it.
✓
Run Instructions In Order Copy each command from the popup and paste it into your computer's Terminal or PowerShell.
1. Login: Connects your terminal to AWS securely. 2. Build: Creates the container image using the Dockerfile. 3. Tag: Labels the image for your specific AWS account. 4. Push: Uploads the file to the ECR registry.
# How the Push Command looks:
docker push <AWS_ACCOUNT_ID>.dkr.ecr.us-east-1.amazonaws.com/credit-default-prediction:latest

Note: I have already fixed your Dockerfile to be compatible with AWS. When you run the "Build" command, it will automatically use the optimized settings I created.

03

Serverless Provisioning

Next, we create an AWS Lambda function that "wakes up" to run your code whenever a request comes in.

✓
Initialization Go to the Lambda Console and click Create function. Select the Container image option.
✓
Image Selection Click Browse images. Select credit-default-prediction and the latest tag.
✓
Configuration (Critical) Under Configuration > General, you MUST set:
- Memory: 1024 MB (needed for model loading) - Timeout: 30 seconds
04

Gateway Configuration

Finally, we create a public URL so users can send data to your AI model.

✓
Deploy HTTP API In API Gateway, create an HTTP API and select your Lambda as the integration.
✓
Invoke URL Copy the Invoke URL provided at the end. This is your API's live address on the web!
